## Body

**Header:** The problem we saw

> A lot of endoscopy and colonoscopy intake still runs on paper, disconnected phone photos, and manually assembled reports — not because clinics don't want better tools, but because the tools that exist assume constant connectivity and a general-purpose workflow that doesn't match this specific procedure.
>
> We built Mein Scope out of Uganda, where unreliable connectivity is an everyday planning constraint, not an edge case. That constraint shaped the product from the start: offline-first wasn't a feature we added later, it's the foundation everything else is built on.

**Header:** What that means in practice

> Patient data lives on the device, not a server you have to reach first. The app installs directly from a browser — no app store review, no IT deployment project. And instead of trying to be a general EHR, Mein Scope is scoped narrowly to endoscopy and colonoscopy workflows, with ENT support planned next.

**Header:** Where we're headed

> Mein Scope today runs on a single device with local storage and manual backup/restore. Multi-device sync and account-based billing are on the roadmap — we're building those deliberately, not rushing them ahead of getting the core clinical workflow right.
